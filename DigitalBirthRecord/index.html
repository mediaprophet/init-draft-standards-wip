This document specifies an ontology for defining agents and related resources in a human-centric manner, emphasizing individual control over personal data and supporting guardianship relations. It redefines "decentralized" as "Human-Centric" while maintaining an innately decentralized methodology.

This is an unofficial draft specification and is subject to change. It is not endorsed by the W3C or any W3C working group.

Introduction

This ontology redefines "decentralized" as "Human-Centric," prioritizing individual control over personal data while maintaining an innately decentralized methodology. It ensures data subjects have a secure personal data store and supports guardianship relations for children and others requiring humanitarian support.

Scope

This ontology defines agents (persons, organizations, software agents), resources (data, services), personal data stores, and guardianship relations to enable human-centric digital identity management.

Terminology

Agent
An entity capable of performing actions, including persons, organizations, and software agents.
Person
A human individual.
Organization
A group of persons working together.
SoftwareAgent
A software program acting autonomously or semi-autonomously.
Resource
Anything owned, controlled, or accessed by an agent, such as data or services.
Data
Information, including personal data.
Service
A service provided by an agent.
PersonalDataStore
A secure storage system for personal data, controlled by a person or their guardian.
Guardianship
A relationship where one agent manages data for another, particularly for those requiring support.
Human-Centric
An approach prioritizing individual control over personal data, redefining "decentralized."

Ontology Specification

The ontology is defined using RDF Schema (RDFS). The namespace is http://example.org/ontology#, abbreviated as ex:.

Classes

  • ex:Agent: An entity capable of actions.
  • ex:Person: A human individual, subclass of ex:Agent.
  • ex:Organization: A group of persons, subclass of ex:Agent.
  • ex:SoftwareAgent: A software program, subclass of ex:Agent.
  • ex:Resource: Anything owned, controlled, or accessed by an agent.
  • ex:Data: Information, subclass of ex:Resource.
  • ex:Service: A service, subclass of ex:Resource.
  • ex:PersonalDataStore: A secure storage for personal data.

Properties

  • ex:hasGuardian: Links a person to their guardian. Domain: ex:Person, Range: ex:Agent.
  • ex:isGuardianOf: Links a guardian to a person. Domain: ex:Agent, Range: ex:Person. Inverse of ex:hasGuardian.
  • ex:owns: Indicates ownership of a resource. Domain: ex:Agent, Range: ex:Resource.
  • ex:controls: Indicates control over a resource. Domain: ex:Agent, Range: ex:Resource.
  • ex:hasDataStore: Links a person to their personal data store. Domain: ex:Person, Range: ex:PersonalDataStore.
  • ex:controlledBy: Indicates the controller of a personal data store. Domain: ex:PersonalDataStore, Range: ex:Agent.
  • ex:storesData: Indicates data stored in a personal data store. Domain: ex:PersonalDataStore, Range: ex:Data.
  • ex:requiresAccessTo: Indicates data required by a service. Domain: ex:Service, Range: ex:Data.
  • ex:hasConsentFrom: Indicates consent for data access. Domain: ex:Service, Range: ex:Agent.

Examples

Below is an example in Turtle notation illustrating the ontology:

ex:Alice a ex:Person ;
  ex:hasGuardian ex:Bob ;
  ex:hasDataStore ex:AlicesDataStore ;

  foaf:name "Bob" .
ex:AlicesDataStore a ex:PersonalDataStore ;

  ex:storesData ex:AlicesData .
ex:AlicesData a ex:Data ;
  ex:ownedBy ex:Alice .
ex:SomeService a ex:Service ;
  ex:requiresAccessTo ex:AlicesData ;
  ex:hasConsentFrom ex:Bob .

Security and Privacy Considerations

Implementations must ensure robust encryption, access controls, and transparent consent mechanisms to protect data subjects. Personal data stores should use secure protocols like TLS and comply with regulations like GDPR.

Implementation Guidelines

Implementations may leverage Solid [[SOLID]] for personal data stores, Decentralized Identifiers [[DID]] for identity management, and RDF-based systems for interoperability. Consent management should align with W3C Data Privacy Vocabularies.

Conclusion

This ontology enables a human-centric approach to digital identity, ensuring data subjects control their data via secure personal data stores and supporting guardianship for humanitarian needs.
